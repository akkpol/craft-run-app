
หมายเหตุสำคัญ
เอกสารนี้เป็น future-state exploration สำหรับคุย flow และออกแบบระบบต่อ ไม่ใช่ runtime source of truth ของแอปปัจจุบัน

ถ้าต้องการดู transition ที่ใช้งานจริงในโค้ดตอนนี้ ให้เช็กเอกสารกลางที่ `docs/WORKFLOW_TRANSITION_TABLE.md` ก่อนเสมอ

ถ้าเนื้อหาในไฟล์นี้ขัดกับ transition table กลางหรือ route handlers ปัจจุบัน ให้ยึด table กลางและโค้ด runtime เป็นหลัก

ได้ครับ อันนี้คือ flow ใหม่แบบไล่ตั้งแต่ต้นจนจบ ให้ครบทั้งลูกค้า ระบบ แอดมิน ทีมออกแบบ ทีมผลิต และการเงิน โดยเผื่ออนาคตที่ระบบเลือก AI/มนุษย์เองได้ด้วย

ภาพรวม
เป้าคือให้ระบบรองรับ 3 เรื่องพร้อมกัน:

ฝั่งลูกค้า: ทักมา ส่งข้อมูล ดูตัวอย่าง อนุมัติ ติดตามงาน

State

ปุ่ม/Action

อันนั้นจะเอาไปทำ database และหน้า admin ต่อได้ตรงมากครับ

ก่อน implement state ใหม่เพิ่มจากเอกสารนี้ ให้เทียบกับ `docs/WORKFLOW_TRANSITION_TABLE.md`, `src/lib/types.ts` และ route handlers ปัจจุบันทุกครั้ง เพื่อลด drift ระหว่างเอกสารกับ runtime
