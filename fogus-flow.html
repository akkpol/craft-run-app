5
ส่งฟอร์ม → สร้าง Lead + Quote
POST /api/intake
LIFF submit ข้อมูลไปที่ API:
— Normalize หน่วย: ซม./ม./นิ้ว/ฟุต → มม. ทั้งหมด
— Upsert customer: สร้างหรืออัพเดทข้อมูลลูกค้า
— สร้าง lead: เก็บทุก field จากฟอร์ม
— คำนวณราคา: ตร.ม. × ราคาต่อวัสดุ + VAT 7%
— สร้าง quote + quote_items พร้อม public_token (32 hex chars)
ถ้าข้อมูลไม่ครบ (ไม่มี due_date หรือขนาด ≤ 0) → ไม่สร้าง quote → สร้าง escalation แทน → ดูหัวข้อ Escalation
FORM_SUBMITTED → WAITING_CUSTOMER_APPROVAL

เปิดหน้าใบเสนอราคา
GET /quote/:token
เปิดใน browser (public URL ไม่ต้อง login) แสดง:
— ข้อมูลลูกค้า: ชื่อ, เบอร์โทร
— รายละเอียดงาน: ประเภท, ขนาด (ซม.), จำนวน, กำหนดส่ง, หมายเหตุ
— รายการ: line items + ราคาต่อรายการ
— สรุปราคา: ก่อน VAT → VAT 7% → รวมทั้งสิ้น (฿)
— วันหมดอายุ (30 วันจากวันสร้าง)

✅ อนุมัติใบเสนอราคา
public_token = 32 hex chars จาก gen_random_bytes(16) → ไม่สามารถ guess ได้
กดอนุมัติ → confirm dialog → POST /api/quotes/:id/approve

Escalation paths · ทางแยก
E1
ลูกค้าขอคุยกับแอดมิน
Escalation
Webhook ตรวจจับ keyword ในข้อความ:
"คุยกับคน" · "คุยกับแอดมิน" · "ขอคุยกับคน" · "ต้องการคุยกับคน" · "admin"

— สร้าง escalation row (reason = ข้อความลูกค้า + conversation_id)
— เปลี่ยน state → HUMAN_REVIEW_REQUIRED
— Bot ตอบ: "ขอบคุณค่ะ ทีมงานจะติดต่อกลับโดยเร็วที่สุดนะคะ 🙏"
— Admin เห็นแถบแดงใน /admin
— Admin ตอบลูกค้าผ่าน LINE OA Manager Chat ได้เลย (Chat + Bot mode ทำงานพร้อมกัน)
HUMAN_REVIEW_REQUIRED
E2
Intake ข้อมูลไม่ครบ
Auto-escalation
ถ้า form submit แต่:
— ไม่มี due_date หรือ
— ขนาด ≤ 0 หลัง normalize

ระบบจะ:
— สร้าง lead (status: new) แต่ ไม่สร้าง quote
— สร้าง escalation: "Incomplete intake data — admin review needed"
— เปลี่ยน state → HUMAN_REVIEW_REQUIRED
— Admin ต้อง review แล้วสร้าง quote เอง (ผ่าน Supabase Dashboard หรือพัฒนาเพิ่มใน phase 2)
HUMAN_REVIEW_REQUIRED
Workflow states · สถานะทั้งหมด
Happy path
NEW_MESSAGE → COLLECTING_INFO → FORM_SUBMITTED → QUOTE_DRAFTED → WAITING_CUSTOMER_APPROVAL → JOB_CREATED → IN_PROGRESS → COMPLETED
Branch: จากทุก state สามารถแยกไป
HUMAN_REVIEW_REQUIRED ← เมื่อ keyword escalation หรือข้อมูลไม่ครบ

  • liff.requestFriendship() fires (check console)
  • Safe-area padding visible on Android (ปุ่ม submit ไม่โดน nav bar บัง)

  • Quote link pushed to customer via LINE push message

  • Approve creates job + job_timeline entry
  • Customer receives LINE notification on approve
  • /status/:token shows current job status + timeline
  • /admin requires login (redirect to /auth/login if not logged in)
  • /admin shows all leads / quotes / jobs / escalations

  • "คุยกับแอดมิน" → creates escalation row + bot replies
